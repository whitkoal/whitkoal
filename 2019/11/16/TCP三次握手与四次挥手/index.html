    TCP三次握手与四次挥手

    计算机网络
    创建时间:2019-11-16 12:59
    阅读:
    1. TCP 概述
    2. TCP报文标志位
    3. 建立连接（三次握手）
    4. 释放连接（四次挥手）

    TCP 概述

    1. TCP是面向连接的运输层协议
    2. 每一条TCP连接只能有一个发送端和一个接收端
    3. TCP是提供可靠交付的服务
    4. TCP提供全双工通信
    5. 面向字节流

    TCP报文标志位

    • URG ：等于1时，表明此段中有紧急数据，需尽快发送
    • ACK ：（确认比特）等于1时，表明此报文是一个确认报文
    • PSH ：等于1时，表示此报文要尽快交给接收进程，不再等接收缓存区填满了再交付
    • RST ：（复位比特）等于1时，表示TCP连接出现严重错误，需重新建立连接
    • SYN ：（同步比特）等于1时，表示此报文是一个协商报文不用放入缓存区，报文是连接请求或是连接接收报文，不存放数据。
    • FIN ：（终止比特）等于1时，表明此报文发送完毕，请求释放连接

    建立连接（三次握手）

    1. 客户端发送连接请求，同步比特SYN=1，序列号SEQ= x
    2. 服务器返回确认，同步比特SYN=1，确认比特ACK=1，序列号SEQ=y, 确认号ack= x+1
    3. 客户端返回确认， 确认比特ACK=1， 序列号SEQ=x+1，确认号ack=y+1

    释放连接（四次挥手）

    1. 主动关闭方请求释放连接：终止比特FIN=1 ， 序列号SEQ = u
    2. 被动方确认：确认标志位ACK=1，序列号SEQ = v，确认号ack = u+1
    3. 被动方应用进程释放连接，再次发送确认：终止比特FIN=1，确认标志位ACK=1，序列号SEQ = w，确认号ack = u+1
    4. 主动方确认关闭：确认标志位ACK=1，序列号SEQ = u+1，确认号ack = w+1

    这时还没结束，因为无法确定被动方是否正确收到最后的确认报文，所以主动方需等待2MSL（最大报文生存时间） ，如果由于信道原因被动方未收到最终确认报文，被动方会重发第3次挥手时所发的报文。
